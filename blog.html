Local storage vs Session storage

Local storage Session storage
This read-only interface property provides access to the Document’s local storage object, the stored data is stored across browser sessions. Similar to sessionStorage, except that localStorage data gets cleared when the page session ends – that is when the page is closed. It is cleared when the last “private” tab of a browser is closed (localStorage data for a document loaded in a private browsing or incognito session). Session Storage objects can be accessed using the sessionStorage read-only property. The difference between sessionStorage and localStorage is that localStorage data does not expire, whereas sessionStorage data is cleared when the page session ends.

Global scope vs Block scope

global scope block scope
Variables declared outside of any function become global variables. Global variables can be accessed and modified from any function. Variables declared inside any function with var keyword are called local variables. Local variables cannot be accessed or modified outside the function declaration.
